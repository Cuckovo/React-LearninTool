> **版本**: 1.0.0 | **更新日期**: 2026-06-15 | **项目路径**: `D:\Github\React-LearninTool`

## 一、项目概述

**LearnTools**（又称「伴学知识库」）是一个**基于 React Native (Expo) 的跨平台高等数学 AI 学习工具**，最初从 [MathLearningTool](https://github.com/Cuckovo/MathLearningTool) 重构而来。

### 一句话定位

> 用 AI 帮你解题 + GeoGebra 帮你画图，一站式高数学习助手。

### 核心理念

```
AI 对话 ──→ 分步解题 & 答疑
    │
    └──→ 自动提取函数表达式 ──→ GeoGebra 实时绘图
              │
              └──→ 将来：构建个人知识库（多学科大纲 + 知识树 + Obsidian 导出）
```

## 三、已完成功能详解

### 3.1 📐 GeoGebra 绘图页面（首页 Tab）

**页面文件**: `src/app/index.tsx`

这是用户打开应用首先看到的页面，嵌入了一个完整的 GeoGebra 图形计算器。

#### 工作方式

| 平台 | 渲染方案 | 文件路径 |
|------|----------|----------|
| **Android** | `react-native-webview` 加载本地文件 | `file:///android_asset/geogebra/GeoGebra.html` |
| **Web** | 原生 `<iframe>` 加载 Metro 静态资源 | `/geogebra/GeoGebra.html` |

#### 关键实现细节

- **加载状态**：页面展示 ActivityIndicator（品牌色 `#90c208` 的转圈动画），加载完成后自动消失
- **错误处理**：WebView 加载失败时显示「加载失败: XXX」中文错误提示
- **AI 跨页面联动**：通过全局状态 `plotCommand` 接收来自 AI 对话页面传入的函数表达式

#### 跨页面通信机制

```
AI 对话页面                          GeoGebra 页面
    │                                     │
    ├─ dispatch(SEND_PLOT_COMMAND) ──────→ plotCommand 状态更新
    │                                     │
    │                              useEffect 检测变化
    │                                     │
    │                          Android: injectJavaScript()
    │                          调用 ggbApplet.evalCommand()
    │                                     │
    │                          Web: iframe.contentWindow
    │                          调用 ggbApplet.evalCommand()
    │                                     │
    │                          5 秒未就绪自动重试
    │                                     │
    │                          dispatch(CLEAR_PLOT_COMMAND)
```

- 支持 **自动重试**：如果 `ggbApplet` 尚未就绪，3 秒（Android）/ 5 秒（Web）后重试
- 对函数表达式中的单引号做了转义处理，防止 JavaScript 注入

### 3.2 🤖 AI 对话页面（第二个 Tab）

**页面文件**: `src/app/ai-chat.tsx`

这是应用的核心交互页面，用户在这里与 DeepSeek AI 进行高等数学对话。

#### 3.2.1 对话功能

- **发送消息**：用户在底部输入框输入数学问题（支持多行），按回车或点击发送按钮
- **加载指示**：等待 AI 回复时显示「AI 正在思考...」+ 旋转动画
- **对话气泡**：用户消息右对齐（品牌绿色背景 + 白色文字），AI 回复左对齐（白色卡片 + 黑色文字）
- **错误提示**：网络错误或 API 异常时，底部显示红色错误条

#### 3.2.2 AI 结构化输出

AI 的回复是**强制格式化**的，包含三部分（由 `src/lib/api.ts` 的系统提示词约束）：

```
【解题过程】
分步解答，公式用 $...$ / $$...$$ 包裹

【图像判断】
可绘制 / 不可绘制

【函数表达式】
f(x) = sin(x)    （如果可绘制）
无                （如果不可绘制）
```

#### 3.2.3 AI 输出解析器

**文件**: `src/lib/ai-parser.ts`

通过正则表达式提取 AI 回复的三个部分：
- `solution`：解题过程文本
- `isPlottable`：是否可绘制（布尔值）
- `functionExpression`：GeoGebra 可用的函数表达式（如 `f(x) = sin(x)`）

函数表达式会经过标准化处理（去掉 LaTeX 语法，确保与 GeoGebra 兼容）。

#### 3.2.4 操作按钮（输入框上方）

当 AI 返回了可绘制的结果后，两个操作按钮会被激活：

| 按钮 | 图标 | 触发条件 | 行为 |
|------|------|----------|------|
| **绘制图像** | GeoGebra 三角图标 | AI 判断「可绘制」 | 传递表达式到 GeoGebra 页面并自动跳转 |
| **分享过程** | 数学语言图标 | AI 返回了解题过程 | 复制解题过程到剪贴板（Web 端） |

未激活状态下按钮呈灰色禁用态。

#### 3.2.5 历史对话管理

- **历史面板**：左侧滑出的抽屉面板，带动画效果（`Animated.timing`，250ms 滑入 / 200ms 滑出）
- **背景遮罩**：点击遮罩可关闭面板
- **会话列表**：显示每个会话的标题（截取前 20 字符）、消息数量、最后更新时间
- **多会话切换**：点击会话即切换，当前活跃会话显示绿色激活点
- **新建会话**：点击右上角「+」按钮创建空白会话
- **删除会话**：长按会话条目可删除（最多保留 10 个会话）

#### 3.2.6 DeepSeek API 集成

**文件**: `src/lib/api.ts`

- **模型**: `deepseek-chat`
- **API 地址**: `https://api.deepseek.com/v1/chat/completions`
- **API Key**: 通过环境变量 `EXPO_PUBLIC_DEEPSEEK_API_KEY` 配置
- **系统提示词**: 严格约束 AI 输出格式（必须包含【解题过程】【图像判断】【函数表达式】三个段落）
- **函数表达式规范**：强制 AI 使用英文函数名（`sin`, `cos`, `log`, `sqrt` 等），禁止在表达式区域使用 LaTeX

### 3.4 全局状态管理

**文件**: `src/lib/app-state.tsx`

使用 React Context + useReducer 实现，无需引入第三方状态库。

#### 状态结构

| 字段 | 类型 | 说明 |
|------|------|------|
| `sessions` | `ChatSession[]` | 所有对话会话列表（最多 10 个），含 type 字段（solver/knowledge） |
| `activeSessionId` | `string \| null` | 当前活跃会话 ID |
| `isLoading` | `boolean` | AI 是否正在响应 |
| `plotCommand` | `GeogebraPlotCommand \| null` | 待传递给 GeoGebra 的绘图命令 |
| `activeTab` | `'geogebra' \| 'ai-chat' \| 'knowledge'` | 当前激活的 Tab |
| `loaded` | `boolean` | 是否已从 AsyncStorage 恢复数据 |
| `activeChatMode` | `'solver' \| 'knowledge'` | 当前对话模式（新增） |
| `activeKnowledgeNodeId` | `string \| null` | 当前学习的知识节点 ID（新增） |

#### 数据持久化

- **存储方式**: expo-sqlite（本地 SQLite 数据库文件 `learntools.db`）
- **ORM 层**: Drizzle ORM（类型安全查询）
- **表结构**: sessions + messages + knowledge_nodes + agent_logs
- **触发时机**: 每次 sessions 变更（消息添加、会话切换/删除）后异步持久化（fire-and-forget）
- **数据库日志**: 通过 `.env` 中 `EXPO_PUBLIC_DB_LOG_LEVEL` 控制级别

#### 派生数据

通过 `getCurrentMessages()` 函数实时计算当前活跃会话的消息列表，作为 `currentMessages` 暴露给组件。

## 七、关键技术细节

### 7.1 GeoGebra 离线方案

- 原始包 115MB，删除 web3d (48MB) 和 webSimple (20MB) 后精简至 48MB
- `GeoGebra.html` 第 14 行 `module = "web3d"` 改为 `"web"` 确保正确加载
- Web 端通过 `app.json` 的 `assetBundlePatterns: ["assets/geogebra/**/*"]` 随 Metro serve
- Android 端通过 `file:///android_asset/` 协议直接读取

### 7.2 AI ↔ GeoGebra 通信

```
【用户输入「求 sin(x) 的导数」】
        │
        ▼
  DeepSeek API 调用
        │
        ▼
  AI 返回：【解题过程】... 【图像判断】可绘制 【函数表达式】f(x) = sin(x)
        │
        ▼
  ai-parser.ts 解析 → { isPlottable: true, functionExpression: "f(x) = sin(x)" }
        │
        ▼
  用户点击「绘制图像」→ dispatch(SEND_PLOT_COMMAND) → router.navigate('/')
        │
        ▼
  GeoGebra 页面 useEffect 检测 plotCommand 变化
        │
        ▼
  injectJavaScript / iframeWindow: ggbApplet.evalCommand('f(x) = sin(x)')
        │
        ▼
  GeoGebra 渲染函数图像 🎉
```

### 7.3 多会话管理

- 每个会话包含 `id`、`title`（截取首条消息前 20 字符）、`messages[]`、时间戳
- 切换到已有会话或新建会话时，`activeSessionId` 变更
- `getCurrentMessages()` 根据 `activeSessionId` 实时计算当前消息列表
- `MAX_SESSIONS = 10`，超出后最早会话被覆盖
- 长按会话条目可删除

## 八、如何运行

### 环境要求

- Node.js ≥ 22
- Expo CLI
- Android Studio（Android 模拟器） 或 真机

### 启动命令

```bash

npm install

# 启动 Expo 开发服务器
npx expo start

npm run web

npm run android
```

需要在项目根目录创建 `.env` 文件（已 `.gitignore`）：

```
EXPO_PUBLIC_DEEPSEEK_API_KEY=sk-xxxxxxxxxxxxx
```
