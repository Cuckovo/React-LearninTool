# 数据库结构文档

## 概述

本项目使用 **expo-sqlite** + **Drizzle ORM** 管理本地数据库，替代原有的 `@react-native-async-storage/async-storage` 单键全量 JSON 方案。

- 数据库文件：`learntools.db`
- ORM 层：Drizzle ORM（`drizzle-orm/expo-sqlite`）
- Schema 定义：`src/db/schema.ts`
- 仓库层：`src/db/repository.ts`

## 表结构

### sessions 表

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | TEXT | PRIMARY KEY | 会话唯一标识（36进制时间戳+随机数） |
| title | TEXT | NOT NULL | 会话标题（取自用户首条消息前20字） |
| created_at | INTEGER | NOT NULL | 创建时间戳（毫秒） |
| updated_at | INTEGER | NOT NULL | 最后更新时间戳（毫秒） |

索引：
- `idx_sessions_updated_at` ON `sessions(updated_at)` — 按更新时间降序查询

### messages 表

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | TEXT | PRIMARY KEY | 消息唯一标识 |
| session_id | TEXT | NOT NULL, FOREIGN KEY | 所属会话 ID，引用 `sessions.id` |
| role | TEXT | NOT NULL | 角色：`user` / `assistant` |
| content | TEXT | NOT NULL | 消息原始文本内容 |
| solution | TEXT | | 解题过程（AI 解析后的 `parsed.solution`） |
| is_plottable | INTEGER | | 是否可绘图：`1`=可绘制，`0`=不可绘制 |
| function_expression | TEXT | | 函数表达式（如 `f(x) = sin(x)`） |
| raw | TEXT | | 原始 AI 回复文本（`parsed.raw`） |
| parsed | INTEGER | NOT NULL DEFAULT 0 | 解析是否成功：`1`=成功，`0`=失败 |
| parse_error | TEXT | | 解析失败时的错误信息 |
| timestamp | INTEGER | NOT NULL | 消息时间戳（毫秒） |

外键：
- `FK_messages_session_id` → `sessions.id` ON DELETE CASCADE

索引：
- `idx_messages_session_id` ON `messages(session_id)` — 按会话查询消息
- `idx_messages_timestamp` ON `messages(timestamp)` — 按时间排序消息

## ER 关系

```
┌──────────┐        ┌──────────┐
│ sessions │ 1 ── N │ messages │
│          │        │          │
│ id (PK)  │◄───────│ session_id│
│ title    │        │ (FK)     │
│ created_ │        │ role     │
│   at     │        │ content  │
│ updated_ │        │  ...     │
│   at     │        │          │
└──────────┘        └──────────┘
     │                    │
     └── ON DELETE CASCADE┘
```

一个会话（sessions）包含多条消息（messages），删除会话时级联删除其所有消息。

## 数据迁移

项目启动时自动检测并执行从 AsyncStorage 到 SQLite 的迁移：

1. 检查 SQLite 是否已有数据 → 有则跳过（幂等）
2. 读取 AsyncStorage 键 `@learntools_sessions` 中的数据
3. 逐条转换并插入 SQLite 表
4. 迁移完成后 AsyncStorage 旧数据保留作为备份

迁移代码位于 `src/db/migrate.ts`。
