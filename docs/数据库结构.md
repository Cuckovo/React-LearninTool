# 数据库结构文档

## 概述

本项目使用 **expo-sqlite** + **Drizzle ORM** 管理本地数据库，替代原有的 `@react-native-async-storage/async-storage` 单键全量 JSON 方案。

- 数据库文件：`learntools.db`
- ORM 层：Drizzle ORM（`drizzle-orm/expo-sqlite`）
- Schema 定义：`src/db/schema.ts`
- 仓库层：`src/db/repository.ts`

## 表结构

### sessions 表

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | TEXT | PRIMARY KEY | 会话唯一标识（36进制时间戳+随机数） |
| title | TEXT | NOT NULL | 会话标题（取自用户首条消息前20字） |
| type | TEXT | NOT NULL DEFAULT 'solver' | 会话类型：`solver`=解题模式，`knowledge`=知识库学习 |
| created_at | INTEGER | NOT NULL | 创建时间戳（毫秒） |
| updated_at | INTEGER | NOT NULL | 最后更新时间戳（毫秒） |

索引：
- `idx_sessions_updated_at` ON `sessions(updated_at)` — 按更新时间降序查询

### messages 表

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | TEXT | PRIMARY KEY | 消息唯一标识 |
| session_id | TEXT | NOT NULL, FOREIGN KEY | 所属会话 ID，引用 `sessions.id` |
| role | TEXT | NOT NULL | 角色：`user` / `assistant` |
| content | TEXT | NOT NULL | 消息原始文本内容 |
| solution | TEXT | | 解题过程（AI 解析后的 `parsed.solution`） |
| is_plottable | INTEGER | | 是否可绘图：`1`=可绘制，`0`=不可绘制 |
| function_expression | TEXT | | 函数表达式（如 `f(x) = sin(x)`） |
| raw | TEXT | | 原始 AI 回复文本（`parsed.raw`） |
| parsed | INTEGER | NOT NULL DEFAULT 0 | 解析是否成功：`1`=成功，`0`=失败 |
| parse_error | TEXT | | 解析失败时的错误信息 |
| timestamp | INTEGER | NOT NULL | 消息时间戳（毫秒） |

外键：
- `FK_messages_session_id` → `sessions.id` ON DELETE CASCADE

索引：
- `idx_messages_session_id` ON `messages(session_id)` — 按会话查询消息
- `idx_messages_timestamp` ON `messages(timestamp)` — 按时间排序消息

## ER 关系

```

│          │        │          │        │                │        │             │
│ id (PK)  │◄───────│ session_id│        │ id (PK)        │        │ id (PK)     │
│ title    │        │ (FK)     │        │ parent_id (FK)──┼──┐     │ session_id  │
│ type     │        │ role     │        │ type           │  │     │ action      │
│ created_ │        │ content  │        │ label          │  │     │ target      │
│   at     │        │  ...     │        │ standard_def   │  │     │ payload     │
│ updated_ │        │          │        │ user_notes     │  │     │ timestamp   │
│   at     │        │          │        │ mastery_status │  │     │             │
└──────────┘        └──────────┘        │ metadata       │  │     └─────────────┘
     │                    │              │ sort_order     │  │

                                          │ updated_at     │  │
                                          └────────────────┘  │
                                               │               │
                                               └── ON DELETE CASCADE (自引用)
```

### knowledge_nodes 表（新增）

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | TEXT | PRIMARY KEY | 节点唯一标识（格式: `kn-{type}-{层级}`） |
| parent_id | TEXT | FOREIGN KEY | 父节点 ID，NULL=根节点。引用 knowledge_nodes.id ON DELETE CASCADE |
| type | TEXT | NOT NULL | 节点类型：`subject` / `chapter` / `section` / `concept` |
| label | TEXT | NOT NULL | 显示名称，如 "1.1.1 映射" |
| standard_definition | TEXT | | 标准定义（教材权威定义，只读） |
| user_notes | TEXT | | 用户自定义笔记（可编辑，1秒防抖自动保存） |
| mastery_status | TEXT | NOT NULL DEFAULT 'not_started' | 掌握状态：`not_started` / `learning` / `passed` / `mastered` |
| metadata | TEXT | | JSON 扩展字段（tags, difficulty, source_session_id 等） |
| sort_order | INTEGER | NOT NULL DEFAULT 0 | 同级排序 |
| created_at | INTEGER | NOT NULL | 创建时间戳（毫秒） |
| updated_at | INTEGER | NOT NULL | 更新时间戳（毫秒） |

索引：
- `idx_nodes_parent` ON `knowledge_nodes(parent_id)`
- `idx_nodes_type` ON `knowledge_nodes(type)`
- `idx_nodes_label` ON `knowledge_nodes(label)`

### agent_logs 表（新增）

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | TEXT | PRIMARY KEY | 日志唯一标识 |
| session_id | TEXT | NOT NULL | 关联会话 ID |
| action | TEXT | NOT NULL | Agent 操作类型：`set_user_notes` / `set_mastery` / `get_node` / `get_children` |
| target | TEXT | | 操作目标（如节点 ID） |
| payload | TEXT | | 操作载荷（JSON 字符串） |
| timestamp | INTEGER | NOT NULL | 操作时间戳（毫秒） |

索引：
- `idx_logs_session` ON `agent_logs(session_id)`

## 数据迁移

项目启动时自动检测并执行从 AsyncStorage 到 SQLite 的迁移：

1. 检查 SQLite 是否已有数据 → 有则跳过（幂等）
2. 读取 AsyncStorage 键 `@learntools_sessions` 中的数据
3. 逐条转换并插入 SQLite 表
4. 迁移完成后 AsyncStorage 旧数据保留作为备份

迁移代码位于 `src/db/migrate.ts`。
